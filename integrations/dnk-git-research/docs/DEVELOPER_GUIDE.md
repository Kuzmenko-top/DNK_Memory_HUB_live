# 💻 Гід для Розробників — DNK Git Research

Цей документ описує внутрішню архітектуру та логіку роботи системи. Призначений для AI-агентів та інженерів, які займаються розвитком продукту.

## 🏗 Архітектура модулів

- `main.py`: Точка входу CLI (Typer). Оркеструє виклики всіх інших модулів.
- `src/search.py`: Взаємодія з GitHub API (PyGithub/httpx).
- `src/analyzer.py`: Логіка AI-аналізу. Використовує Gemini 2.0 Flash для генерації структурованих звітів.
- `src/ranker.py`: Алгоритм ранжування. Розраховує `dnk_total_score` та `integration_friction`.
- `src/storage.py`: Управління даними (JSON Library + Qdrant Vector DB).
- `src/report.py`: Генерація Markdown звітів та таблиць.
- `core/prompts/`: Централізоване сховище AI-промптів.

## 📊 Формати Даних

### library.json
Центральний файл знань. Кожен об'єкт репозиторію містить:
- Метадані GitHub.
- AI Dossier (what_it_does, core_features, etc.).
- Розраховані оцінки та вердикти.

### catalog.md
Markdown-представлення бібліотеки. Використовується для швидкого перегляду людиною. Оновлюється автоматично при додаванні нових репо.

## 🧬 Логіка оцінювання (Scoring Logic)

### Integration Friction Score (0–10)
Вираховується в `src/ranker.py` (або безпосередньо в промпті аналізу):
- `+2.0`: наявність Dockerfile/docker-compose.
- `+1.5`: наявність .env.example.
- `+1.0`: наявність тестів.
- `-1.0`: за кожні 10 залежностей (понад 5).
- `-2.0`: копілефт ліцензія (GPL).

## 🚀 Інтеграція з іншими агентами

### FastMCP Server
Модуль `mcp_server/` реалізує протокол FastMCP. Це дозволяє будь-якому іншому агенту (наприклад, Hermes) викликати інструменти пошуку та аналізу репозиторіїв безпосередньо через інтерфейс інструментів (tools).

### Integration Recipes
Рецепти генеруються у форматі, оптимізованому для "проковтування" AI-кодерами (Claude Code). Вони мінімізують токенний бюджет, вказуючи лише на найважливіші файли.
