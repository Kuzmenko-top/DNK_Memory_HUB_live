## Правила

### Дедуплікація

- Один репозиторій = один рядок
- Ідентифікатор: `owner/name` (lowercase)
- Якщо є — оновити, ніколи не дублювати

### Валідація

- Fit: 0.0–10.0 (ніколи > 10)
- Friction: 0.0–10.0 (ніколи > 10)

### Новий репозиторій

1. Перевірити чи є в таблиці
2. Якщо немає → додати ⚪ QUEUED, Fit/Friction = `—`
3. Після аналізу → заповнити, статус → 🟢 ANALYZED

### Автозаповнення

Блок `catalog_entry` з кінця звіту автоматично заповнює рядок таблиці.

## Цикл переаналізу (кожні 4 дні)

### Пріоритет

1. 🔵 INTEGRATED — найвищий (наш код залежить)
2. ⚪ QUEUED — новий, потрібен перший аналіз
3. 🟢 ANALYZED — рутинна перевірка
4. ❌ NOT RECOMMENDED — раз на 30 днів

### Що перевіряти

**🔵 INTEGRATED:**

- Нові коміти, зміна ліцензії, архівування, breaking changes, security advisories
- Сумісність з нашим кодом
- Якщо breaking changes → завдання на допрацювання

**🟢 ANALYZED:**

- Свіжість, ліцензія, архівування, major release
- Якщо major release → повний переаналіз
