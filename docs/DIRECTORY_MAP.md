# 🗺 Directory Map — DNK_HUB Ecosystem

Цей документ визначає структуру робочого простору, призначення кожної директорії та правила взаємодії для AI-агентів та розробників.

## 🏛 Коренева структура (The Architecture)

| Директорія | Призначення | Режим доступу | Правила Engagement |
| :--- | :--- | :--- | :--- |
| `.antigravity/` | Системні інструкції агента | Read-Only (для AI) | Містить "душу" та правила поведінки основного AI-помічника. |
| `core/` | **Engine Room.** Ядро системи, оркестратори та боти. | Restricted | Жодних кастомних скілів тут. Тільки двигуни та фундаментальна логіка. |
| `infra/` | **The Foundry.** Інфраструктура, скрипти, конфіги лінтерів. | System | Тут живуть інструменти підтримки "Швейцарського годинника". |
| `integrations/` | **The Satellites.** Готові продукти та зовнішні сервіси. | Product | Кожна підпапка — це ізольований продукт (напр. `dnk-git-research`). |
| `memory/` | **The Brain.** Векторна пам'ять та Obsidian Vault. | Strategic | Найцінніший актив. Дані мають бути структурованими та портативними. |
| `plugin/` | **The Skills.** Кастомні розширення для агентів. | Custom | Тут живе твоя унікальність. Скіли, воркфлоу та кастомні промпти. |
| `docs/` | **The Archive.** Архітектура, стандарти та плани. | Knowledge | Вся документація системи. |
| `projects/` | **The Workshops.** Специфічні бізнес-проєкти. | Workspace | Місце для розробки нових продуктів. |

## 🛡 Протоколи Стабільності (Stability Protocols)

1. **Zero-G Root Policy**: Корінь має залишатися чистим. Усі нові файли мають знайти своє місце в існуючій структурі.
2. **Isolation First**: Зовнішній код ніколи не змішується з кастомною логікою.
3. **Relative Paths Only**: Жодних абсолютних шляхів. Система має запускатися з будь-якої папки.
4. **Self-Healing**: Будь-яка зміна структури має супроводжуватися запуском `infra/scripts/self_check.sh`.
